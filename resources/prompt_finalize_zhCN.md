# 完成任务

**重要：这是自动化流程的最后一步。**
- 没有人工介入——只需完成收尾工作
- 不要提问或请求批准——直接执行步骤
- 直接且实事求是——不需要对话式语言

任务循环已完成。现在请完成收尾工作：

## 你的操作

1. **检查 git 状态**
   - 你当前在哪个分支？
   - 是否有未提交的更改？如果有，请提交它们。
   - 是否有要推送的提交？

2. **推送功能分支（如果适用）**
   - 如果你创建了功能分支并有提交，请推送它：`git push -u origin <分支名>`
   - 如果推送失败（无远程仓库、认证问题），请记录并继续

3. **创建 PR（如果适用）**
   - 仅在以下情况创建：在功能分支上、推送成功、`gh` CLI 可用
   - 检查 PR 是否已存在：`gh pr list --head <分支名>`
   - 如果不存在 PR，请创建一个具有良好标题和描述的 PR
   - 包含更改内容和测试方法

4. **返回原始分支**
   - 切换回起始分支

5. **输出摘要**
   - 列出已完成的事项
   - 注明任何问题或未完成的项目
   - 如果创建了 PR，请包含 PR 的 URL
